Additional Blacklist Support

Below are assorted versions of Steven Black's unified hosts files reformatted for various other applications for additional support and are synced regularly. Feel free to open an issue if you have any suggestions to support another format, or contact me via the "Contact ScriptTiger" link on the left side of the page.

FQDN
Just a standard list of domains, which is the most common blacklist format used by DNSCrtypt, pfSense, etc. If for some reason you need an FQDN list but this one is not working for you, please don't hesitate to open an issue or contact me so we can figure it out and get it resolved for the benefit of yourself and everyone else.

RFQDN
Similar to the FQDN list of domains except that children of parent domains that are present on the list are pruned from the list. This is useful if you would like a reduced FQDN list of domains to be blocked and all child domains are already inherently blocked by the intended software.

RPZ
Designed to be used with DNS RPZ, but does not include the zone file header elements, these may be different from user to user and must be appended to the top of each zone file depending on your individual preferences.

RRPZ
Similar to the RPZ format except that multiple child domains of a single blocked parent domain are reduced to a single wild card child domain of the shared common domain. Please note if multiple child domains of a common parent domain are present but the parent domain itself is not, those entries will not be reduced as the parent itself is not blocked and may have other child domains that shouldn't be blocked, as well.

VRPZ
Also similar to the RPZ format except that all blocked domains receive an associated wild card to block all child domains, whether or not child domains are or are not explicitly blocked or not, following Paul Vixie's recommended approach. Child domains of parents domains are then also removed from the list to reduce redundant entries.

IPv4_IPv6
Includes both IPv4 lines leading with "0.0.0.0" and IPv6 lines leading with "::" for devices that are compatible with the standard hosts file format but require both IPv4 and IPv6 entries. Normally if a domain is found, the system should take the result whether it is IPv4 or IPv6, but in some systems this is not the case and require explicit entries for both.

Compressed
An alternative hosts file format which condenses multiple domain names and comments to a single line. If your are having issues with the regular format hosted by Steven Black or in the Hosts Packages section of this site, you can try this and see if it works for you. Some Windows users suffering from poor performance when using the regular format have noticed dramatic performance gains when using this format. This format may also be better for some mobile applications.

MCompressed
The same as the compressed format except it's only halfway compressed to a 5-domain maximum per line instead of the 9 domains per line in the fully compressed format. Some users have found this balance to work better on their systems.
